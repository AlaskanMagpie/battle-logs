# Doctrine (`/doctrine`) — cards, binder, catalog, units

Use this skill whenever you add or change **doctrine content**: structures, commands, card PNG/SVG art, unit spawn profiles, tower meshes, binder codex membership, or quick-match defaults.

## Mandatory commands (execute, do not hand-wave)

1. **Card files** (`public/assets/cards/*.{png,webp,jpg,jpeg,svg}`) or `public/assets/cards/manifest.json` behavior  
   From repo root: **`npm run cards:pipeline`**  
   That regenerates `manifest.json`, bumps `CARD_ART_CACHE_BUSTER` in `src/ui/cardArtManifest.ts` when the card map changes, and runs binder catalog tests.

2. **Unit / tower GLBs** (`public/assets/units/*.glb`) or `animationProfiles` in `public/assets/units/manifest.json`  
   From repo root: **`npm run assets:sync-manifest`**  
   `*.glb` is Git LFS — stage those paths with `git add` in the same commit.

3. **Ship check**  
   **`npm run build`** (runs card manifest sync + `vite build`).

## Naming (failure mode #1)

- **Catalog id** = `StructureCatalogEntry.id` / `CommandCatalogEntry.id` in `src/game/catalog.ts` (snake_case, stable).
- **Card file stem** must match that id: `public/assets/cards/{catalogId}.{ext}`.
- **`producedUnitId`** must match an `animationProfiles[].id` in `public/assets/units/manifest.json` (auto-generated by `npm run assets:sync-manifest` from GLB families). You may point a **semantic** constant at an existing profile id (e.g. lava monks → `lanternbound_line`) until a dedicated `lava_*` family exists on disk.
- **Tower mesh**: override in `src/render/glbPool.ts` `TOWER_GLB_OVERRIDES` when the hashed pick is wrong; filename must exist in `manifest.json` `files` (URL-encode spaces in dev tools if needed).

## Where doctrine surfaces

| Concern | Primary files |
|--------|----------------|
| Stats / costs / signals | `src/game/catalog.ts`, `src/game/constants.ts` |
| Binder codex grid + quick-picks | `src/ui/binder/DoctrineBinderPicker.tsx` (`FULL_ART_STRUCTURE_CARD_IDS`, `QUICK_PICK_IDS`) |
| Quick match default row | `src/game/quickMatchDoctrine.ts` |
| Stat overlay positions on printed cards | `src/ui/cardArtOverlayLayouts.json` |
| Tower + unit GLB routing | `src/render/glbPool.ts`, `public/assets/units/manifest.json` |
| Structure footprint (battle scale) | `structureDims` in `src/render/scene.ts` (signal + class rules) |

## Agent checklist

1. Add or replace assets on disk (`public/assets/cards/`, `public/assets/units/`).
2. Wire `catalog.ts`, manifest profiles, and any `TOWER_GLB_OVERRIDES` / binder lists.
3. Run **`npm run cards:pipeline`** if cards changed; **`npm run assets:sync-manifest`** if GLBs changed.
4. Run **`npm test`** and **`npm run build`** before merge.

## Replaces

- **`cards-binder`** — that skill is retired; use **`doctrine`** for all card + binder + catalog + unit work.
